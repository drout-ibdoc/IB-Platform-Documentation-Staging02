Partial Lock Preferences Beta

Note: This feature is newly introduced in Platform version 6.7.0.0 and currently supports Objects and Object Types only.

This feature aims to improve the partial locking feature by allowing users to configure components with partial lock preferences for better access control. These improvements, however, continue to be in developmental stages and are not recommended for production use.

Additionally, fully locked or partially locked components will not be visible to protect sensitive code, enhancing security within installed tenants. It is recommended that users test these features in a controlled environment.

The More Actions menu on the application setup page includes a Partial Lock Preferences option. For every component supporting partial locking, you can select Unlocked, Partially Locked or Locked radio buttons.

When any installed application is generated with it's lock status set to Partially locked, configurations this page is applied. The components can be further updated by the user before the application XML is generated. For components currently not supporting partial lock, have only Locked and Unlocked status. For any other lock statuses apart from Partially locked, the configurations saved to the Partial Lock Preferences page does not apply.

For more information on generating application XML, see Generate XML. For more information on generating application XML using REST API 2.0, see Retrieve Application Definition.

Managing Partial Lock Permissions

  • If the installed application is Fully Locked, the partial lock preferences page is no longer visible to the tenants. Hence, it is impossible to generate XML.

  • If the installed application is Partially Locked, the tenant can view the partial lock preferences page, without having any access to modify the page. Yet, the application XML can be generated with the for further redistribution or change management purposes.

  • If the installed application is Unlocked, the tenant is free to modify the partial lock preferences page and generate the application XML as required.

Security Enhancements along with Partially Locked Applications

Note: This section is applicable to both partially locked and fully locked components.

It is important to note that this change in Partial Locked Preferences was made to enhance security measures and protect important files from being accidentally or maliciously altered by unauthorized users. A tenant when installs the partially locked application can perform the following actions on the components after installing the partially locked application by the administrator. This applies to Object Script, Formula, Template, Expression Fields, Custom Method, Hosted File, Button, Workflow Action and Trigger.

For example: When a hosted file attached to any partially locked or fully locked application, the content of the hosted file is not visible to the user.

The following sections details the behavior of the actions under every component.

Applications

Based on application's lock status, the tenant is allowed to make changes as described in the following table:

Action Unlocked Partially Locked Locked
Clone Allowed Allowed Not Allowed
Edit Allowed Allowed Allowed - limited
Integration Name Allowed Not Allowed Not Allowed
Add Tabs Allowed Allowed Not Allowed
Reorder Tabs Allowed Allowed Not Allowed
Remove Tabs Allowed Only Customizable & Configurable Tabs Not Allowed
Delete Allowed Allowed Allowed

Objects

Based on object's lock status, the tenant is allowed to make changes as described in the following table:

Action Unlocked Partially Locked Locked
Clone Object Allowed Allowed Allowed
Create Relationship Between This Object & Another Allowed Allowed Allowed
Delete Object Allowed Not Allowed Not Allowed
Undeploy Object Allowed Allowed Not Allowed
Change Label Allowed Allowed Not Allowed
Change Description Allowed Allowed Not Allowed
Change Integration Name Allowed Not Allowed Not Allowed
Record Name Template Allowed Allowed Not Allowed
Add Attributes Allowed Not Allowed Not Allowed
Remove Attributes Allowed Not Allowed Not Allowed
Add Properties Allowed Allowed - Changing Audit Trail is also allowed Not Allowed
Remove Properties Allowed Allowed Not Allowed
Create Child Components Allowed Allowed Not Allowed

Clone Child Components Allowed Allowed Not Allowed
Delete Child Components Allowed Only child components created by customer (not in source app) can be deleted. Not Allowed
Note:

  • Creating Relationship Between This Object & Another is now allowed when the application XML is locked.

    • Object Type

    • Chart

    • Gauge

    • View

    • Relationship
